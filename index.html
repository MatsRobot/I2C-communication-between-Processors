🖧 I2C-Bridge: Multi-Processor Data Exchange

A specialized hardware-to-hardware link bridging the Raspberry Pi Pico (Master) and Arduino (Slave). This physical data bridge allows for modular scaling by offloading sensor polling and ultrasonic distance tracking to a dedicated secondary processor.

Physical Link Integrity: By establishing a common ground and a master-driven clock (SCL), this system ensures reliable data transfer across the physical wire-link, bypassing the typical software lag of high-level protocols.

Wired Link & Level-Shifter Logic Architecture

How It Works: The Hardware Interconnect

This implementation treats the I2C bus as a dedicated physical link between disparate architectures (ARM Cortex-M0+ and ATmega328P):

  • Physical Interconnect: Uses a 4-wire setup (VCC, GND, SDA, SCL) to create a persistent data pipeline.
  • Bus Isolation: The RP2040 uses its secondary I2C1 channel to isolate high-bandwidth OLED traffic from the critical sensor-link on I2C0.
  • Level Shifting: Managing the bidirectional 3.3V ↔ 5.0V interface to ensure logic level safety across the link.

Hardware Interrupts

The link uses hardware-level onRequest interrupts on the Arduino side, meaning data is served immediately without the Master needing to wait for a sensor-poll loop.

Modular Scaling

This "Bridge" design allows additional Arduino "Link-Nodes" to be added to the same SDA/SCL lines by simply assigning unique hex addresses.
