I2C Communication Between Processors

This project explores reliable I2C data transfer in readable text format between different microcontrollers. The primary focus is establishing a Pico as Master and an Arduino as Slave, while navigating the limitations of MicroPython's I2C Slave libraries.

The Technical Challenge: While I2C theoretically supports multiple masters, synchronizing clocks (SCL) is a major hurdle. Furthermore, using a Pico as a slave in MicroPython is difficult due to limited library support for assigning slave addresses.

Pico (Master) to Arduino (Slave) Setup

In this working model, a Raspberry Pi Pico acting as Master uses I2C0 (Pins GP16/GP17) to communicate with an Arduino and an OLED Display on the same bus.

  • Arduino Slave Address: 10 (0x0A Hex)
  • OLED Address: 60 (0x3C Hex)
  • Sensor: MaxSonar Ultrasound transducer connected to Arduino.

Handling Bus Conflicts

A key finding in this project was the management of the data bus. While the Pico can write to the OLED, the Arduino (as slave) cannot communicate with the OLED on the same bus without creating interrupt conflicts. The documentation within the code details how to disable specific initiations to maintain bus stability.
